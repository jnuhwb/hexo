iTunes Connect提交，Invalid Binary问题

发表于 2015-07-03 | 分类于 iOS

新版提交AppStore审核，第一次遇到Invalide Binary的问题，记录备忘。

iTunes Connect会把Binary的状态显示在Prerelease里，以前没出问题就一直没注意到。

Prerelease里没有错误提示，其实苹果是通过发邮件提示错误，不明白苹果为何不能把这个结果直接显示在iTunes Connect上。

Dear developer,

We have discovered one or more issues with your recent delivery for “app名字”. To process your delivery, the following issues must be corrected:

Non-public API usage:
The app references non-public symbols in xxx: UICreateCGImageFromIOSurface

最近接入Reveal，Reveal使用了苹果私有接口，只能在Debug版本添加。一开始是用CocoaPod，当时指定了Debug，但是经常连接不上，后来改用静态库接入，配置的时候就忘了只在Debug配置。

Select the Build Settings tab and add the following to the Debug configuration of the Other Linker Flags setting:

用strings确认新包没用引入Reveal，重新上传iTunes Connect。上传后二进制状态一直是Uploaded，几个小时都没变化，猜想可能是上传太多次了（我上传了5次），苹果有限制吧，不过没查到相关的文档说明，等到第二天，终于变成了Inactive？不是Processing??不过Build可以选择了，不管了，提交审核先，有知道为什么是Inactive的朋友求告知。
